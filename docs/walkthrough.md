# Walkthrough

## 개요
이 문서는 2026-03-11 기준으로 `DL` 폴더 프로젝트에 대해 무엇을 어떻게 수정했는지 단계별로 정리한 작업 기록입니다.

## 1. 초기 상태
초기 프로젝트는 `DL.py`에 UI, 계통 생성, 시뮬레이션 루프, 제어 로직이 섞여 있었습니다. 이 상태에서는 기능 추가와 유지보수가 어렵고, 자동 분석/보고서/알고리즘 수정 기능을 확장하기에 구조가 불안정했습니다.

## 2. 구조 분리
### 반영 내용
- `sim_engine.py` 추가
- `coordinated_engine.py` 추가
- `limit_finder.py` 추가
- `DL.py`는 UI 중심으로 재구성

### 이유
- UI와 계산 엔진 분리
- 단일 시나리오와 자동 분석 재사용성 확보
- 보고서 생성과 알고리즘 고도화 작업 분리

## 3. 버그 수정 및 안정화
### 엑셀 다운로드 후 화면 초기화 문제
- `st.session_state`를 사용해 결과와 다운로드 바이트를 유지하도록 수정
- 다운로드 후 rerun이 발생해도 결과 화면이 사라지지 않게 정리

### 실행 중단 후 상태 꼬임 문제
- 우상단 Stop으로 중단했을 때 다음 rerun에서 자동 재실행 루프에 빠지지 않도록 복구 로직 추가
- `재실행`, `초기화` 제어를 추가해 사용자 복구 흐름 정리

### 실행 중 설정 변경 문제
- 시뮬레이션 중에는 사이드바, 계통 모델, 시간패턴, 알고리즘 탭 입력을 잠그도록 수정
- 사용자에게 잠금 안내 문구 출력

## 4. 협조제어 고도화
### 기본 운영 조건
- 전압 허용범위: `0.94 ~ 1.06 p.u.`
- 선로용량 한도: `12 MVA`
- OLTC 1대, ESS 1대 구조

### 반영 내용
- 상태기반 협조제어 적용
- 선로 혼잡과 전압 이상을 함께 평가
- OLTC 우선 + ESS 보조 제어 구조 반영
- ESS 효율을 SOC 계산에 반영
- 제어 파라미터를 UI에서 수정 가능하게 구성

## 5. 자동 민감도 분석 기능 추가

## 11. 연구형 배치 시나리오 생성기 추가
### 배경
이전 batch 구조는 `PV x ESS x Load` 식 조합 중심이라 연구 질문과 직접 연결되지 않는 경우가 있었습니다. 그래서 무작위/무의미 조합을 줄이고, 질문 중심으로 시나리오를 생성하도록 구조를 바꿨습니다.

### 새 원칙
- UI는 유지하고 기존 `Batch Scenario Runner` 안에서만 동작
- 무작위 조합 금지
- 의미 없는 Cartesian product 금지
- 연구 질문별 `scenario mode`로 생성
- 생성과 실행을 분리

### 지원 mode
- `hosting_capacity`
- `load_pv_map`
- `ess_sizing`

### mode별 동작
- `hosting_capacity`: Load, ESS, control을 고정하고 PV만 단조 증가시켜 첫 위반 시점과 수용 한계를 찾음
- `load_pv_map`: Load와 PV를 목적성 있는 2D 운전점으로 배치해 운영영역을 작성함
- `ess_sizing`: 대표 스트레스 케이스를 고정하고 ESS 크기 및 위치를 바꿔 효과를 비교함

### 생성 흐름
- 사용자 설정 또는 CLI 입력 수집
- mode별 settings 구성
- 고정 변수/가변 변수 검증
- 불가능한 조합 제거
- 중복 제거
- `SCN_###` ID, label, description 부여
- preview 표 생성
- 기존 batch 실행 엔진에 전달

### 예시 진행
- `hosting_capacity`: `SCN_001 PV 0.80 -> SCN_002 PV 1.00 -> SCN_003 PV 1.20`
- `load_pv_map`: `SCN_001 Load 0.90 / PV 0.80, SCN_002 Load 0.90 / PV 1.20, SCN_003 Load 1.10 / PV 0.80, SCN_004 Load 1.10 / PV 1.20`
- `ess_sizing`: `SCN_001 Size 0.00 / Bus 5 -> SCN_002 Size 0.50 / Bus 5 -> SCN_003 Size 1.00 / Bus 5`

### 구현 포인트
- `coordinated_engine.py`: mode 정의, generator, validation/filter, scenario metadata, preview helper, batch summary metadata
- `DL.py`: 기존 expander 안에 mode 선택과 preview 연결
- `limit_finder.py`: batch mode CLI 인자 확장
- 보고서: 시나리오 진행 원칙과 예시 흐름 텍스트 추가

## 12. 데이터 관리/보고서/상태 동기화 추가 개선
### 데이터 관리 효율화
- `build_all_scenarios_excel_bytes`를 통해 자동 민감도 분석의 모든 회차 상세 시계열을 단일 Excel로 병합
- 배치 실행에서도 요약(`CSV/Excel`) + 상세(`Excel`)를 함께 내려받을 수 있도록 연결
- `st.data_editor` 편집값을 세션에 즉시 병합하는 동기화 보강으로 입력 유실/더블 편집 이슈 완화
- 모선 수 증감 시 기존 입력값이 사라지지 않도록 `resize_bus_dataframe_preserve_values`로 값 보존 리사이즈 적용

### 상태 관리 및 UX 안정화
- `ESS 설치 버스` 선택 후 다시 기본값으로 돌아가던 문제를 세션 정규화 로직 수정으로 해결
- 모선 개수 변경값이 재실행 시 5로 복귀하던 문제를 정규화 정책 수정으로 해결
- `권장 기본값 적용(Bus 5개)`에서 widget 초기화 충돌이 나던 문제를 `pending_cfg_update` 기반 적용으로 해결
- 사용자 설정 저장을 `cfg_*`뿐 아니라 `bus_df`, `time_df`까지 확장해 앱 재시작 후에도 상태를 복원

### 보고서 품질 개선
- run-by-run 섹션에 각 case별 `ESS SOC` 시계열 그래프 추가
- combined overlay 섹션에 `ESS SOC by Run` 그래프 추가
- 샘플 보고서 형식 일관성을 위해 그래프 포함 fallback DOCX 생성 경로를 우선 사용

### 전압 그래프 왜곡 원인 및 완화
- 원인: 기존 그래프는 각 시점의 전역 min/max 전압을 envelope로 그려, 최악 버스가 바뀌는 지점에서 선이 급격히 꺾여 보이는 artifact가 발생
- 완화 1: 보고서 그래프를 “대표 저전압 버스 + 대표 고전압 버스” 연속 시계열로 재구성
- 완화 2: ESS P/Q 명령을 즉시 목표값으로 점프시키지 않고 ramp 추종으로 변경하여 고배율 구간 파형의 급격한 변형을 완화
